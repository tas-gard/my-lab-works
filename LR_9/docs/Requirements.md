# Требования к сдаче лабораторной работы

1. Наличие программы в одном/нескольких файлах(задание к ЛР смотрите в методичке к оной ЛР)
2. Соответствие стиля кода CodeStyleGuide.md, который лежит в этой же директории(для ЛР на языке программирования Си)
3. Наличие отчета в pdf формате(шаблон отчета прилагается)
4. Ваше решение должно располагаться в ветке solution. Именно в таком виде. Любые другие названия веток проверяться не будут.
5. Ваше решение должно располагаться в папке solution/. Изменять файлы в других папках изменять не надо. Работы, где это правило нарушается проверяться не будут.

		Если вы случайно изменили файл, но не сделали коммит, то все изменения можно отменить командой `git stash`
		Если коммит был сделан, то его можно откатить командой `git revert`
6. Отчет должен быть в формате pdf. Word-отчет можно перегнать в формат pdf, через раздел печати(Печать -> печать в pdf).
7. Отчет и приложение должен располагаться в папке solution/report/
8. Имя отчета должно быть задано по следующему шаблону:
		
		Для ЛР: ЛР<номер ЛР числом>_<Фамилия, инициалы без точек>_<номер группы числом>_<номер в списке числом>.pdf
		Для ЛР: ПриложениеЛР<номер ЛР числом>_<Фамилия, инициалы без точек>_<номер группы числом>_<номер в списке числом>.pdf
		Для КП: КП<номер ЛР числом>_<Фамилия, инициалы без точек>_<номер группы числом>_<номер в списке числом>.pdf
		Для КП: ПриложениеКП<номер ЛР числом>_<Фамилия, инициалы без точек>_<номер группы числом>_<номер в списке числом>.pdf

	Нормер ЛР/КП соответствует номеру в **старом** списке.
	
	Имя отчета и приложения к нему на примере меня:
	
		ЛР9_БучкинТА_401_1.pdf
		ПриложениеЛР9_БучкинТА_401_1.pdf
	
	Внутренняя структура отчета не меняется

9. В папке solution/ должен располагаться Makefile, который собирает ваш проект. Для ЛР9 я сделаю этот файл сам и на ближайшей паре расскажу, как им пользоваться в дальнейших лабах 

10. В последнем коммите должно быть следующее сообщение "Work done". Без этого сообщения ЛР не будет считаться готовой к проверке. Для коммита нужны измененные файлы, так что если вы забыли отсавить в последнем коммите нужное сообщение, то можно добавить в любой тестовый файл пустую строку(например) и сделать новый коммит с нужным сообщением

		git commit -m "Work done"

11. В папке solution/ должны располагаться только файлы, относящиеся к решению вашей работы, т.е. отчет, приложение к нему, Makefile, и файлы с кодом. Все остальные файлы(артефакты от Word, файлы Zone.indentifier от Windows, бинарные файлы скомпилированных программ) должны быть удалены.
12. Одобрение Pull Request'a в gitVerse

Итоговая структура файлов папки solution:

solution/

|-- report/

|--|-- report/<Отчет>

|--|-- report/<Приложение к отчету>

|-- Makefile

|-- main.c

|-- src/

|--|-- src/*.c

|-- include/

|--|-- include/*.h

# Оценивание
На лабораторную работу выделяется время в 2 недели, после этого каждую неделю максимальная оценка будет снижаться вплоть до 3 при сдаче после 3-х и более недель с момента выдачи лабораторной работы.

|Дата выдачи|Дедлайн на 5|Дедлайн на 4|
|-----------|------------|------------|
|16.11.2025|30.11.2025|07.12.2025|

	Датой сдачи считается момент времени, когда был сделан последний коммит, после которого ваш PR
	получил одобрение
	!После сдачи ЛР для утверждения оценки требуется очное собеседование со мной по теме ЛР
# Требования к допуску к зачету

Для допуска к зачету необходимо:
1. Сдать все лабораторные работы **вместе с отчетами**
2. Предоставить отчет по практике с отметками о всех сданных ЛР за семестр
	1. Не забудте проставить свои оценки и даты сдачи в таблицу работ.
